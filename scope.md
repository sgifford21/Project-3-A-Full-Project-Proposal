# Project Scope Statement  
## The Locker Room – Online Storefront Concept

---

## Project Purpose | Feedback Loop

**Why are we doing this project?**  
The purpose of this project is to create a professional two-page website that represents my former business, The Locker Room, which I operated while stationed at Clay Kaserne in Germany. This project will showcase my experience as a business owner, highlight customizable apparel products, and demonstrate both creativity and project management skills.

**Who needs to provide feedback? When and how?**  
Feedback will primarily come from the course instructor and peers. Feedback will be provided through assignment reviews and discussion posts. Adjustments will be made after each major milestone, including the initial scope submission, website development, and final review.

---

## Description | People Resources

**How would you describe this project?**  
This project involves designing and building a two-page website using HTML and CSS that simulates an online storefront. The website will include a product gallery, custom order information, a contact section, and a location concept connected to Clay Kaserne. The design will be simple, professional, and easy to navigate.

**Who are the key team members?**  
- **Project Manager:** Susan Gifford  
- **Developer/Designer:** Susan Gifford  
- **Stakeholder:** Course Instructor  
- **Users:** Military families, soldiers, and potential future customers

---

## Desired Results | Priorities

**What specific outcomes must we achieve to be successful?**  
- Create a functional two-page website using `index.html` and `about.html`
- Include a product gallery showcasing customizable apparel
- Provide clear custom order and contact information
- Apply consistent styling using one external CSS file
- Demonstrate professionalism, organization, and creativity
- Successfully publish the website through GitHub Pages

**How would you prioritize these constraints?**  
- **Time:** Project must be completed by the course deadline
- **Quality:** Website must be clear, functional, and professionally presented
- **Budget:** No financial budget; only free tools will be used

Priority order: **Time → Quality → Budget**

---

## Constraints | Risks

**What limitations or restrictions will impact this project?**  
- **Time Constraint:** Project must be completed by the course deadline
- **Technical Constraint:** Project is limited to beginner-level HTML and CSS skills
- **Resource Constraint:** Only free tools will be used, including GitHub and Visual Studio Code
- **Scope Constraint:** Project is limited to a two-page website per course requirements

**What risks could impact success?**  
- Learning curve with coding may slow progress
- GitHub publishing or formatting issues may delay final submission
- Time management conflicts may reduce time available for testing and revision

---

## Exclusions | Acceptance Criteria

**What is out of the project scope and should NOT be included?**  
- No e-commerce functionality or real purchases
- No user accounts or login systems
- No backend or database integration
- No advanced JavaScript or animations

**Who needs to sign off on the project and when?**  
- **Stakeholder:** Course Instructor
- **Approval Date:** Final Project 03 submission deadline
